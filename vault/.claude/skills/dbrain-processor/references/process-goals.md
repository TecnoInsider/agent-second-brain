# Process Goals

## Concept

**Outcome goals** focus on WHAT you want to achieve:
- "Заработать $3K/мес"
- "Запустить BaZi-сервис"
- "Переехать в Валенсию"

**Process goals** focus on HOW you do it:
- "1 BaZi-ролик в день"
- "2 сессии по 2 часа на калорий-бот в неделю"
- "Написать 1 человеку в день с предложением бесплатной расшифровки"

## Why Process Goals Work Better

Based on research (meta-analysis of 27 experiments in sports):

1. **Контроль** — Ты контролируешь действия, не результат. «Продать расшифровку» создаёт тревогу; «написать 1 человеку» — понятно и выполнимо.
2. **Уверенность** — Процессные цели превращают большую задачу в маленькие шаги, повышая самоэффективность.
3. **Фокус** — Ежедневная ясность: что делать, а не абстрактное «надо бы...».

**Особенно важно для Вадима:** Склонность к перфекционизму (баг Петуха) делает outcome-цели токсичными — «запустить продукт» вызывает паралич «ещё не готово». Process-цели обходят этот баг: ты просто делаешь шаг, не оценивая готовность.

## How to Transform

| Outcome Goal (ПЛОХО) | Process Goal (ХОРОШО) |
|----------------------|----------------------|
| "Запустить BaZi-сервис" | "2 часа работы над дашбордом каждое утро" |
| "Набрать 1000 подписчиков" | "2–3 AI-ролика + 1 YouTube видео в неделю" |
| "Начать зарабатывать на BaZi" | "1 бесплатная расшифровка в неделю + снять видео про процесс" |
| "Закрыть кредитки" | "Каждый платёжный день — оплатить минимум по всем картам" |
| "Пройти курс Анарбаевой" | "1 урок курса 3 раза в неделю" |
| "Выучить BaZi" | "1 обучающий ролик в день" |
| "Добить калорий-бот" | "2 сессии по 2 часа с помощницей в неделю до 28 февраля" |
| "Переехать в Валенсию" | "1 час в неделю: поиск квартир / расчёт бюджета / чеклист документов" |

## Patterns for Process Goals

### Time Blocks
Regular dedicated time for deep work:
- "2 часа работы над BaZi-дашбордом каждое утро 8:00–10:00"
- "1 час BaZi-обучение после обеда"
- "30 мин контент (пост/ролик) перед обедом"

### Frequency Commitments
Consistent action cadence:
- "1 BaZi-ролик в день (обучение)"
- "2–3 AI-ролика в неделю (контент)"
- "1 бесплатная расшифровка в неделю (кейсы + сарафан)"
- "1 YouTube видео в неделю"

### Checkpoints
Review milestones:
- "Воскресенье: 15 мин недельный обзор"
- "Конец месяца: обзор прогресса по целям"
- "Показать черновик оффера 1 человеку до пятницы"

### Streaks
Daily habits:
- "Зарядка + контрастный душ каждое утро"
- "Отбой до 23:00"
- "1 BaZi-ролик в день — не пропускать"

## When Processing Voice Entries

If user says outcome goal, transform to process:

```
User: "Надо начать продавать BaZi-консультации"

BAD: Create task "Начать продавать BaZi-консультации" p2
→ Размыто, вызывает тревогу, непонятно что делать

GOOD: Create tasks:
1. "Написать 1 знакомому с предложением бесплатной расшифровки" p3 due:tomorrow
2. "Набросать черновик оффера: что входит + цена (30 мин)" p3 due:friday
→ Конкретно, выполнимо, контролируемо
```

```
User: "Хочу закончить калорий-бот"

BAD: Create task "Закончить калорий-бот" p2
→ Слишком большое, парализует

GOOD: Create task "Сессия с помощницей: демо-режим часть 1 (2 часа)" p2 due:tomorrow
→ Один конкретный шаг, ограничен по времени
```

```
User: "Нужно больше подписчиков на канале"

BAD: Create task "Набрать подписчиков" p3
→ Outcome, не контролируется

GOOD: Create task "Записать быстрый AI-ролик с экрана (10 мин)" p3 due:today
→ Действие, которое 100% в твоём контроле
```

## For Weekly Planning

When generating Process Commitments for ONE Big Thing:

1. **Ask:** What daily/weekly habits would make this inevitable?
2. **Limit:** 3–5 process commitments max (правило фокуса!)
3. **Prioritize:** Time blocks > Frequency > Checkpoints
4. **Create:** Recurring tasks in Todoist with appropriate patterns
5. **Check:** Не создаёт ли это растекание? (баг Свиньи) Лучше 3 процессных цели, которые выполняются, чем 7 которые забрасываются.

## Recurring Task Patterns

Map process goals to Todoist dueString:

| Process | dueString |
|---------|-----------|
| "каждое утро" | "every day at 7:30" |
| "каждый день" | "every day" |
| "в рабочие дни" | "every weekday" |
| "3 раза в неделю" | "every monday, wednesday, friday" |
| "раз в неделю" | "every week" |
| "к пятнице" | "friday" (one-time) |
| "2 раза в неделю" | "every tuesday, thursday" |

## Anti-Patterns

Avoid creating process goals that are:
- Too vague: "Работать над проектом" (сколько? когда? над чем конкретно?)
- Too ambitious: "4 часа deep work 5 дней в неделю" (нереалистично → бросишь через неделю)
- Untrackable: "Думать о стратегии" (не измеримо)
- Too many: 7+ процессных целей одновременно (растекание → баг Свиньи)

Good process goals are:
- **Конкретны** в действии (что именно делаю)
- **Ограничены** по времени/частоте (сколько минут, сколько раз)
- **Отслеживаемы** (сделал / не сделал — бинарно)
- **Посильны** (правило 80%: лучше делать 80% каждый день, чем 100% раз в месяц)

Based on: Meta-analysis of 27 experiments in sports showing process goals significantly outperform outcome/performance goals.
